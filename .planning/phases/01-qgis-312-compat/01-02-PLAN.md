<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GDAL raster input → numpy array | User-supplied raster data enters inference pipeline |
| onnxruntime ONNX model file → inference | Model loaded from local disk via InferenceSession |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ONNX model file (Google-Resnet101.onnx) | accept | Model loaded from local plugin models/ directory; no network fetch; file integrity is user's responsibility |
| T-02-02 | Information Disclosure | GDAL raster data read into RAM | accept | Local file processing only; no PII; data never leaves the machine |
| T-02-03 | Denial of Service | Large raster causes OOM in band.ReadAsArray() | accept | Known existing bottleneck (CONCERNS.md performance section); not in scope for this phase |
| T-02-04 | Elevation of Privilege | onnxruntime model execution | accept | onnxruntime runs in CPU mode with no system privilege escalation; ONNX models cannot execute arbitrary code in onnxruntime's inference engine |
</threat_model>
